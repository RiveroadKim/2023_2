### 데이터 베이스의 구조

```
스키마
-데이터베이스에 저장되는 데이터 구조의 제약조건
인스턴스
-스키마에 따라 데이터베이스에 실제로 저장된 값

3단계 데이터베이스 구조
External level - 개별 사용자 관점
Conceptual level - 조직 전체의 관점
Internal level - 저장 장치의 관점
```


#### 데이터 독립성
```
-하위 스키마를 변경하더라도 상위 스키마가 영향을 받지 않는 특성

논리적 데이터 독립성
-개념 스키마가 변경되어도 외부 스키마는 영향을 받지 않음
-변경되면 관련된 외부/개념 사상만 정확하게 수정

물리적 데이터 독립성
-내부 스키마가 변경되어도 개념 스키마는 영향을 받지 않음
-변경되면 관련됨 개념/내부 사상만 정확하게 수정
```

```
ㅁ데이터 사전 = 시스템 카탈로그(System catalog)
메타 데이터를 유지하는 시스템 데이터베이스(메타 데이터 - 데이터에 대한 데이터)

스키마, 사상 정보, 다양한 제약조건 등을 저장
데이터베이스 관리 시스템이 "스스로 생성하고 유지"함
일반 사용자도 접근이 가능 but 저장 내용을 검색만 가능

ㅁ데이터 디렉터리
데이터 사전에 있는 데이터에 실제로 접근하는데 필요한 위치 정보를 저장하는 시스템 데이터베이스
일반 사용자의 접근은 허용되지 않음

ㅁ사용자 데이터베이스
사용자가 실제로 이용하는 데이터가 저장되어 있는 일반 데이터베이스
```

### 데이터베이스 사용자
```
ㅁ데이터베이스 사용자
데이터베이스를 이용하기 위해 접근하는 모든 사람

데이터베이스 관리자 - 데이터벧이스 시스템을 운영하고 관리
최종 사용자 = 일반 사용자 - 데이터베이스에 접근하여 데이터를 조작
응용 프로그래머 - 데이터 언어를 삽입하여 응용 프로그램을 작성

ㅁ데이터베이스 관리자(DBA; DataBase Administrator)
데이터베이스 시스템을 운영 및 관리하는 사람
주로 데이터 정의어와 데이터 제어어를 이용
주요 업무
• 데이터베이스 구성 요소 선정
• 데이터베이스 스키마 정의
• 물리적 저장 구조와 접근 방법 결정
• 무결성 유지를 위한 제약조건 정의
• 보안 및 접근 권한 정책 결정
• 백업 및 회복 기법 정의
• "시스템" 데이터베이스 관리
• 시스템 성능 감시 및 성능 분석
• 데이터베이스 재구성

ㅁ최종 사용자(End user)
데이터베이스에 접근하여 데이터를 조작(삽입,삭제,수정,검색 -> 문제가 생겼으면 갱신이상) 하는 사람
주로 데이터 조작어를 이용
캐주얼 사용자와 초보 사용자로 구분

ㅁ응용 프로그래머(Application programmer)
데이터 언어를 삽입하여 응용 프로그램을 작성하는 사람
주로 데이터 조작어를 이용
```

### 데이터 언어
```
ㅁ데이터 언어
-데이터 정의어(DDL; Data Definition Language)
  스키마를 정의, 수정, 삭제를 위해 이용
  ex) create table, alter table, drop table...

-데이터 조작어(DML; Data Manipulation Language)
  데이터의 삽입, 삭제, 검색 등의 처리를 요구하기 위해 사용
  ㅁ절차적 데이터 조작어(procedural DML)
    사용자가 어떤(what) 데이터를 원하고 그 데이터를 얻으려면 어떻게(how) 처리해야 하는지도 설명
  ㅁ비절차적 데이터 조작어(nonprocedural DML)
    사용자가 어떤(what) 데이터를 원하는지만 설명
    선언전 언어(decalarative language)라고도 함

-데이터 제어어(DCL; Data Control Language)
  내부적으로 필요한 규칙이나 기법을 정의하기 위해 사용
★사용 목적
– 무결성 : 정확하고 "유효한" 데이터만 유지
– 보안 : 허가받지 않은 사용자의 데이터 접근 차단, 허가된 사용자에게 권한 부여
– 회복 : 장애가 발생해도 데이터 일관성 유지
– 동시성 제어 : 데이터 동시 공유 지원
```

### 데이터베이스 관리 시스템의 구성
```
주요 구성 요소
• 질의 처리기(query processor)
– 사용자의 데이터 처리 요구를 해석하여 처리
– DDL 컴파일러, DML 프리 컴파일러, DML 컴파일러, 런타임 데이터베이스 처리기, 
"트랜잭션 관리자" 등을 포함
• 저장 데이터 관리자(stored data manager)
– 디스크에 저장된 데이터베이스와 데이터 사전을 관리하고 접근
```
